# Skills (Agent Skills в Cursor)

## Что это?

**Skills (Agent Skills)** в Cursor — это markdown-файлы (`SKILL.md`), которые содержат инструкции для AI-агента по выполнению конкретных задач. Скилл описывает пошаговый сценарий, критерии и примеры, чтобы агент действовал единообразно в заданной области (например, создание правил, ревью кода, работа с PDF).

## Зачем нужны?

### 1. Единые воркфлоу

Один и тот же сценарий можно применять в разных чатах и проектах без повторения инструкций вручную.

### 2. Переиспользуемые инструкции

Скилл упаковывает экспертизу (форматы, чек-листы, шаблоны) в один файл, который агент подключает по необходимости.

### 3. Личные и проектные скиллы

- **Личные** (`~/.cursor/skills/`) — доступны во всех ваших проектах (ваши привычки, стиль коммитов, любимые инструменты).
- **Проектные** (`.cursor/skills/` в репозитории) — общие для команды (стандарты ревью, процесс деплоя, доменная логика).

### 4. Когда применять

Скилл уместен, когда нужно:
- задать пошаговый сценарий (создать правило, скилл, миграцию);
- зафиксировать формат вывода (коммит-сообщения, отчёты);
- передать доменные знания, которые агент не знает по умолчанию.

## Как работают?

### Где хранятся

| Тип       | Путь                    | Область видимости      |
|----------|-------------------------|------------------------|
| Личный   | `~/.cursor/skills/`     | Все ваши проекты       |
| Проектный| `.cursor/skills/`       | Только этот репозиторий|

Скилл — это **папка** с обязательным файлом `SKILL.md` внутри. Остальные файлы (например, `reference.md`, `examples.md`, скрипты) опциональны.

### Как агент выбирает скилл

Агент смотрит на поле **description** в начале каждого `SKILL.md`. По нему решается, подходит ли скилл к запросу пользователя. Поэтому описание должно ясно говорить: *что* делает скилл и *в каких случаях* его применять.

## Структура SKILL.md

Каждый скилл обязан содержать:

1. **YAML frontmatter** в начале файла:
   - `name` — короткий идентификатор (латиница, цифры, дефисы; до 64 символов).
   - `description` — краткое описание для агента (до 1024 символов): что делает скилл и когда его использовать.

2. **Тело в Markdown** — заголовки, инструкции, примеры. Рекомендуется разделы вроде «Инструкции», «Примеры», при необходимости ссылки на дополнительные файлы (reference.md, examples.md).

Пример минимальной структуры см. в [example-structure.md](./skills/example-structure.md).

## Skills и Rules — в чём разница?

| | Rules | Skills |
|---|--------|--------|
| **Файлы** | `.mdc` в `.cursor/rules/` | `SKILL.md` в `.cursor/skills/` или `~/.cursor/skills/` |
| **Назначение** | Постоянный контекст: стандарты кода, стиль, соглашения по проекту или по типам файлов | Разовые или повторяемые сценарии: «создай правило», «напиши коммит», «проведи ревью по чек-листу» |
| **Когда применяются** | Автоматически при открытии файлов (по glob) или всегда (alwaysApply) | Когда запрос пользователя совпадает с description скилла |

Rules задают *как* писать код в проекте; Skills — *как* выполнить конкретную задачу по шагам.

## Связь с LLM и MCP

Skills работают поверх LLM и при необходимости поверх MCP:

- **LLM** выполняет инструкции из скилла и генерирует ответы.
- **MCP** даёт агенту доступ к инструментам (файлы, API, БД); скилл может описывать, когда и как эти инструменты использовать.
- **Skills** настраивают поведение агента: какие шаги выполнять, в каком формате отвечать, какие критерии соблюдать.
