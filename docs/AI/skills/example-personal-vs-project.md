# Личный и проектный скилл

Скиллы бывают **личными** (доступны во всех ваших проектах) и **проектными** (лежат в репозитории и доступны всем, кто с ним работает).

## Где хранятся

| Тип | Путь | Кто видит |
|-----|------|-----------|
| Личный | `~/.cursor/skills/имя-скилла/SKILL.md` | Только вы, во всех проектах |
| Проектный | `.cursor/skills/имя-скилла/SKILL.md` в корне репозитория | Все, кто клонирует репозиторий |

Важно: каталог `~/.cursor/skills-cursor/` зарезервирован за встроенными скиллами Cursor; свои скиллы туда не создают.

## Когда использовать личный скилл

- Привычки и стиль, которые вы хотите применять везде: формат коммитов, структура заметок, любимые линтеры.
- Утилиты, не привязанные к одному репо: работа с PDF, конвертеры, личные шаблоны отчётов.
- Эксперименты: можно держать скилл у себя и не засорять общий репозиторий.

Пример условного пути личного скилла:

```
~/.cursor/skills/commit-msg/
└── SKILL.md
```

## Когда использовать проектный скилл

- Общие стандарты команды: как проводить ревью, какой формат коммитов или PR.
- Процессы, привязанные к проекту: деплой, миграции, генерация кода из доменных правил.
- Онбординг: новый разработчик получает те же скиллы вместе с репозиторием.

Пример условной структуры в репо:

```
my-project/
├── .cursor/
│   └── skills/
│       └── code-review/
│           └── SKILL.md
├── src/
└── ...
```

## Краткое сравнение

| Критерий | Личный | Проектный |
|----------|--------|-----------|
| Область видимости | Все ваши проекты | Один репозиторий |
| Версионирование | Только у вас (например, отдельный репо или бэкап) | В Git вместе с проектом |
| Команда | Только вы | Все, кто клонирует репо |
| Типичное содержание | Стиль, личные утилиты, эксперименты | Стандарты команды, процессы проекта |
