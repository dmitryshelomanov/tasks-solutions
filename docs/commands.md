# Команды Cursor: добавление задачи

В Cursor доступна команда **Process Task** (Обработать задачу). Запуск: Command Palette (`Cmd+Shift+P` / `Ctrl+Shift+P`) → введите «Process Task» и выберите команду. Можно передать путь к файлу с задачей в параметрах; иначе используется текущий открытый файл.

Команда обрабатывает файл с алгоритмической задачей и приводит его к стандартам проекта: определяет категорию, очищает код, добавляет описание и теги, пишет тесты, переименовывает и перемещает файл в нужную папку.

---

## Что делает команда

1. **Определяет категорию задачи** и папку для размещения:
   - `easy-top-50/` — задачи из топ-50 легких LeetCode
   - `sliding-window/` — задачи на скользящее окно
   - `binary-search/` — задачи на бинарный поиск
   - `graph/` — задачи на графы
   - `javascript/` — задачи по JavaScript (LeetCode JavaScript challenges)
   - `yandex/` — задачи от Яндекса
   - Корень `tasks/` — прочие задачи

2. **Очищает код**:
   - Удаляет `console.log`, `console.error`, `debugger`
   - Удаляет неиспользуемые импорты и переменные
   - Удаляет закомментированный код (кроме альтернативных решений, показывающих разные подходы)
   - Удаляет отладочный код и временные переменные

3. **Добавляет структуру по стандартам проекта**:
   - Ссылка на оригинал (если есть)
   - JSDoc с описанием задачи, подхода и сложности
   - Теги паттернов (см. список ниже)
   - Комментарии в коде на русском

4. **Пишет тесты** на vitest с основными и граничными случаями.

5. **Переименовывает файл** в формат `{номер}. {Название}.test.js` или `{Название}.test.js`.

6. **Перемещает файл** в выбранную папку категории.

## Примеры оформления

### Пример: Two Sum

```javascript
// https://leetcode.com/explore/interview/card/top-interview-questions-easy/92/array/546/

/**
 * Найти два числа, сумма которых равна target
 *
 * Задача: Найти индексы двух чисел в массиве, сумма которых равна target.
 * Каждый вход имеет ровно одно решение, нельзя использовать один элемент дважды.
 *
 * Паттерны: #arrays, #hashtable
 */

import { expect, test } from "vitest";

var twoSum = function (nums, target) {
  const map = new Map();
  for (let i = 0; i < nums.length; i++) {
    if (map.has(target - nums[i])) {
      return [map.get(target - nums[i]), i];
    }
    map.set(nums[i], i);
  }
};

test("twoSum", () => {
  expect(twoSum([3, 3], 6)).toEqual([0, 1]);
  expect(twoSum([2, 7, 11, 15], 9)).toEqual([0, 1]);
  expect(twoSum([3, 2, 4], 6)).toEqual([1, 2]);
});
```

## Важные правила

1. **Комментарии и описания** — на русском языке.
2. **Теги** — формат `#tag_name` (с решёткой, без пробелов).
3. **Сложность** — в формате O(n), O(n²), O(log n) и т.д.
4. **Задачи из LeetCode** — номер в начале названия файла (например, `146. LRU Cache.test.js`).
5. **Тесты** — полное покрытие основных и граничных случаев.
6. **Код** — без отладочных логов и неиспользуемого кода.
